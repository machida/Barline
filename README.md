# Barline

バンド練習用のリードシートをブラウザだけで作成・印刷できる静的サイトです。

## 起動

`index.html` をブラウザで開いてください。サーバーや外部ライブラリは不要です。

## テスト

Node.js標準のテストランナーを使います。外部パッケージは不要です。

```bash
node --test tests/app.test.js
node --check app.js
```

テストでは、JSON正規化、BPM正規化、保存失敗時の描画継続、コード表記ゆれの自動変換、プレビュー行分割と幅配分、転調・拍子変更の表示、コード枠の折り返し、ディグリー表記、同梱フォント参照、Coda/セーニョ/to Coda、ページ途中のセクションマーク非表示、セクション終端の二重線を確認します。

実装の全体像、state構造、行分割、ページ送り、SVG座標、JSON互換仕様は `ARCHITECTURE.md` を参照してください。AI/人間向けの作業前提は `CLAUDE.md` にまとめています。

## 主な機能

- 曲名、アーティスト、Key、BPM、拍子の編集
- ♯優先／♭優先の調号表記切り替え
- プレビューだけを半音単位で一括移調
- セクション追加、コピー、削除、左右への並べ替え
- セクションのメモ
- 小節単位の転調、拍子変更
- 1小節内のコード入力枠を追加・削除して、簡易的なリズム位置を表現
- リピート、1カッコ、2カッコ、任意文字のカッコ、to Coda、Coda、セーニョ、Fine、D.C.、D.S.
- データの書き出し・読み込み
- 編集内容のリセット
- localStorageへの自動保存
- A4縦印刷

## フォント

UIとコードなどの通常テキストは `Noto Sans JP` / `Noto Sans` を優先して表示し、未インストール環境ではシステムフォントへフォールバックします。楽譜内のト音記号、調号、拍子記号、Codaなど音楽記号寄りの文字要素は同梱した `Bravura` を使います。ディグリー表記のローマ数字は同梱した `Noto Serif` を使います。`to Coda`、`Fine`、`D.C.`、`D.S.` のような方向指示の文字部分には、同梱した `Noto Serif` を方向指示専用 face として使います。拍子記号の数字は通常の `0-9` ではなく、BravuraのSMuFL拍子数字グリフで描画します。BravuraとNoto SerifはSIL Open Font License 1.1で配布されているため、`assets/bravura-OFL-1.1.txt` と `assets/noto-serif-OFL.txt` を同梱しています。外部フォントの読み込みには依存しないため、オフラインでも動作します。

## 調号表記

曲情報の「調号表記」で選択します。

- `♯ 優先`: 黒鍵のKeyと移調後コードを `C#`、`D#`、`F#` などで表示
- `♭ 優先`: 黒鍵のKeyと移調後コードを `D♭`、`E♭`、`G♭` などで表示

Key選択肢はメジャー／マイナーに分け、半音順に並びます。

調号表記を切り替えても、入力済みコード文字列は書き換えません。未移調のコードはユーザーが入力した表記を保持し、移調プレビューで生成される音名だけが `♯ 優先` / `♭ 優先` に従います。

## 一括移調

「プレビューを移調」をオンにして、`-11` から `+11` の移調幅を選択します。

- 元のコードやKeyは変更しません
- 印刷プレビュー、調号、コード、小節転調Keyへ反映します
- JSONへ移調オン／オフと移調幅を保存します

## ディグリー表記

「ディグリー表記」をオンにすると、プレビュー上のコード横にKey基準のディグリーを併記します。例えばKeyがCで `C7` と入力している場合は `C7(I7)` と表示します。移調プレビューがオンの場合は、移調後のKeyとコードを基準にディグリーを表示します。

## 編集UI

左カラムは「曲情報」と「セクション」のタブで切り替えます。

- 「曲情報」では曲名、アーティスト、Key、BPM、拍子、調号表記、プレビュー移調、ディグリー表記を編集します
- 「セクション」ではセクション、メモ、小節、コード、オプションを編集します
- ヘッダーの「書き出し」「読み込み」はJSONファイルを扱います
- 「リセット」は現在の編集内容とlocalStorageの保存内容を初期状態へ戻します。実行前に確認ダイアログを表示します

## セクション

- タブで編集するセクションを切り替えます
- `←` / `→` でセクションを並べ替えます
- セクション名の右側に並べ替え、コピー、削除ボタンがあります
- セクションメモはセクション名の下に表示します

## 小節とコード

各小節には標準で `1` から `4` のコード入力欄があります。`＋ コード枠` で枠を増やすと、プレビュー上の小節を入力欄の数で等分し、入力位置に合わせてコードを表示します。何も入力しない枠はブランクとして扱うため、例えば「コード、空欄、コード、コード」のような簡易的なリズム位置を作れます。

「オプション」を開くと、リピートなどの記号、転調、拍子変更を設定できます。閉じている状態でも、設定済みの項目は要約表示されます。

- 「転調」をオンにすると、その小節左の小節線が二重線になり、小節内の左側に新しい調号を表示します。その小節以降のKeyも変わります
- 「拍子変更」をオンにすると、その小節左の小節線が二重線になり、小節内の左側に新しい拍子記号を表示します。その小節以降の拍子も変わります
- 「任意カッコ」に文字を入力すると、1カッコ/2カッコと同じ位置にその文字を表示します。入力がある場合は1カッコ/2カッコより優先します
- 元のKey・拍子へ戻す場合は、戻したい小節で転調・拍子変更を設定します
- セクションの開始小節は二重線になります。リピート開始の場合はリピート記号を優先します
- 基本は1段4小節ですが、転調や拍子変更、コード枠の増加で小節内のコード表示領域が狭くなる場合は、1段3小節などへ自動調整します
- 最終段や自動調整された段で4小節未満の場合、存在しない小節分の五線は表示しません
- A4に収まらない場合は段とセクション見出しを考慮して複数ページへ分割し、プレビューにページ番号を表示します。セクション途中で紙送りされた場合、見間違い防止のため続きページにはセクションマークを表示しません

コード入力欄は、よくある表記ゆれを自動で整えます。

- 全角英数記号を半角へ変換します
- `#` は `♯` へ変換します
- `b` は音名直後なら `♭` へ変換します
- スラッシュ前後の空白を削除します
- 小文字のルート音を大文字へ変換します
- `A-7`、`A−7`、`Aー7` は `Am7` へ変換します
- `C△7`、`C▲7` は `Cmaj7` へ変換します
- `CM7`、`CM9`、`CM13`、`CM6`、`CMaj7`、`CMAJ7` は `Cmaj7` などへ変換します
- `C○7`、`C〇7`、`C°7` は `Cdim7` へ変換します
- `C+7` は `Caug7` へ変換します
- `Cø` は `Cm7♭5` へ変換します

既存JSONの単一 `chord` 形式も読み込めます。新しい形式では次のように保存します。

`position` は後方互換のため保存していますが、現在のプレビュー描画は `position` の数値を直接使いません。1小節内のコード枠数で小節幅を等分し、枠の順番に沿ってコードを配置します。読み込み時も `position` は枠順に合わせて再採番されます。

```json
{
  "chords": [
    { "text": "Gm", "position": 0 },
    { "text": "F", "position": 0.25 },
    { "text": "E♭", "position": 0.5 },
    { "text": "D", "position": 0.75 }
  ]
}
```

## JSONの追加項目

```json
{
  "accidentalPreference": "flat",
  "degreeNotation": true,
  "transpose": {
    "enabled": true,
    "semitones": 2
  },
  "sections": [
    {
      "label": "B",
      "bars": [
        {
          "key": "E♭",
          "timeSignature": "3/4",
          "endingText": "3",
          "chords": []
        }
      ]
    }
  ]
}
```

旧形式のセクション直下にある `key` と `timeSignature` は、読み込み時に先頭小節へ自動移行します。
